# Examples

Two short walkthroughs for getting a feel of the TRAS pipeline without
spending API credits.

## 1. Run the offline debug pipeline

The debug config uses an in-process fake LLM (`DebugModel`) that returns
deterministic synthetic answers. The full TRAS pipeline — forward rollout,
textual gradient, textual regularizer, MCSA aggregation, update, MCTS
search — runs end-to-end in about two minutes without any API keys.

```bash
NO_WANDB=1 poetry run python src/main.py -c configs/debug.yaml
```

Swap `mcts` for `tras` in the config to label the run accordingly:

```bash
NO_WANDB=1 poetry run python - <<'PY'
import yaml, subprocess, tempfile, pathlib
cfg = yaml.safe_load(pathlib.Path("configs/debug.yaml").read_text())
cfg["search_algo"] = "tras"
tmp = pathlib.Path(tempfile.mkstemp(suffix=".yaml")[1])
tmp.write_text(yaml.safe_dump(cfg))
subprocess.run(["poetry", "run", "python", "src/main.py", "-c", str(tmp)])
PY
```

## 2. Inspect a completed run

Every run writes a timestamped log directory under `./logs/<ts>-<task>-algo_<name>/`
containing:

| File                 | What it contains                                                           |
| -------------------- | -------------------------------------------------------------------------- |
| `metrics.jsonl`      | Structured per-phase metrics (one JSON object per line)                    |
| `tree_report.html`   | Live HTML visualization of the MCTS tree (auto-served if `open_report: true`) |
| `data.json`          | Final tree, all optimized prompts, and the selected best                   |
| `plots/`             | Matplotlib figures generated by the reporter                               |

Quick inspection with `inspect_run.py` (reads the latest run under `logs/`
and prints a summary):

```bash
poetry run python examples/inspect_run.py
```
